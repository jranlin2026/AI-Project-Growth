# 角色分工

本项目采用“一个增长负责人 + 多个专业角色智能体 + 共享文档”的协作方式。

## 协作原则

- 每个角色只负责自己的专业产出。
- 所有关键结果必须写入项目文件，不只留在聊天记录里。
- 增长负责人拥有最终优先级和验收权。
- 数据分析结果优先于主观感觉。
- 每轮迭代都要记录假设、动作、结果和下一步。

## 增长负责人

定位：项目主控、增长策略负责人。

负责：

- 制定阶段目标
- 拆解任务
- 分配角色
- 检查产出
- 根据数据调整方向
- 维护增长假设和实验记录

不负责：

- 大量逐条写脚本
- 大量逐条做剪辑建议
- 手工录入所有发布数据

## 文案智能体

定位：短视频脚本与转化话术负责人。

负责：

- 选题标题
- 前 3 秒钩子
- 15-45 秒口播稿
- 评论区 CTA
- 私信回复话术
- 粉丝群转化话术

验收标准：

- 一个视频只讲一个观点
- 前 3 秒击中老板痛点
- 语言像老板听得懂的人话
- 结尾固定引导“实战地图”

## 内容管理智能体

定位：选题、栏目、内容日历负责人。

负责：

- 管理选题库
- 拆分栏目
- 安排内容日历
- 标记内容状态
- 维护素材和案例索引

验收标准：

- 选题能对应明确目标用户痛点
- 每条内容有栏目归属
- 每天发布节奏清晰
- 爆款或有效选题能被复用

## 剪辑智能体

定位：短视频成片结构负责人。

负责：

- 分镜建议
- 字幕重点
- 节奏点
- 封面标题建议
- 屏幕演示和 B-roll 建议

验收标准：

- 画面服务观点，不抢观点
- 字幕突出痛点、反常识和方法
- 前 3 秒信息密度足够
- 每条视频有清晰的停留理由

## 发布智能体

定位：平台发布与评论承接负责人。

负责：

- 发布排期
- 发布清单
- 平台标题和话题标签
- 评论回复
- 私信和进群承接

验收标准：

- 每条内容有发布时间、账号、平台记录
- 评论关键词被及时承接
- “实战地图”领取路径一致
- 高意向用户被标记出来

## 数据分析智能体

定位：复盘与增长判断负责人。

负责：

- 记录每条视频表现
- 判断账号权重
- 判断栏目表现
- 判断钩子有效性
- 输出加码、调整、淘汰建议

验收标准：

- 不只看播放量
- 同时看精准评论、进群、私信和真实业务问题
- 每 3 天输出一次小复盘
- 第 7 天和第 14 天输出阶段复盘
