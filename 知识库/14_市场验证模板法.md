# 市场验证模板法

## 一句话结论

短视频选题和文案不能只靠内部想法，必须先找市场已经验证过的线索成交型内容模板，再用 AI 拆结构、换行业、换痛点、换交付，生成适合本项目的拍摄脚本。

## 为什么要这么做

当前最大风险不是没有内容，而是内容自嗨。

自嗨型内容的典型表现：

1. 我们觉得观点很对，但用户前 3 秒没有停留理由。
2. 脚本体系完整，但不像市场上用户会主动评论的内容。
3. 内容讲方法论很多，但没有线索成交逻辑。
4. 只讲 AI 概念，没有直接击中老板的获客、销售、客服、成本问题。

市场验证模板法的目标是：

```text
市场已经证明什么能吸引精准用户
  -> AI 拆它的成交结构
  -> 换成 AI 商业落地场景
  -> N哥用口语化方式拍出来
```

## 判断参考内容是否值得拆

| 判断项 | 看什么 |
| --- | --- |
| 评论区 | 有没有人问“怎么做”“怎么拿”“适合什么行业” |
| 用户身份 | 评论里是否像老板、创业者、业务负责人 |
| 线索意图 | 有没有领取资料、咨询方法、询问合作的动作 |
| 痛点强度 | 是否围绕获客、销售、客服、内容、成本、管理 |
| 可迁移性 | 能不能改写成 AI 商业落地场景 |

不追泛爆款，追精准线索。播放和点赞只是参考，评论、私信、进群和真实业务问题才是核心。

## 拆模板，不抄文案

可以拆：

1. 开头如何筛选精准人群。
2. 中间如何否定错误做法。
3. 如何给出简单可执行方法。
4. 如何把评论引导到资料领取。
5. 评论区为什么有人行动。

不能做：

1. 逐句搬运原文。
2. 冒充别人案例。
3. 使用不属于我们的行业数据或承诺。
4. 承诺收益、保证成交、保证爆单。

## 标准工作流

```text
找市场参考
  -> 判断评论区是否有线索
  -> 拆母模板结构
  -> 替换成 AI 商业落地场景
  -> 生成 3 个开头
  -> 生成 60-90 秒主版脚本
  -> 生成 30-40 秒短版脚本
  -> 给 N哥拍摄口语提示
  -> 发布后看评论/私信/进群/真实业务问题
  -> 有效则沉淀成可复用母模板
```

## 母模板拆解表

每条待拍选题必须先填这个表。

| 字段 | 说明 |
| --- | --- |
| 参考来源 | 对标视频、同行账号、相邻行业内容、客户真实问题 |
| 市场证据 | 评论区是否有咨询、领取资料、业务问题 |
| 目标人群 | 这条内容筛选哪类老板 |
| 原始痛点 | 用户原本为什么停留 |
| 可迁移痛点 | 换成 AI 商业落地后是什么痛点 |
| 开头结构 | 原视频前 3 秒怎么抓人 |
| 中段结构 | 原视频如何给方法或制造信任 |
| CTA 结构 | 原视频如何引导评论、私信、进群 |
| 我们的改写方向 | 改成哪条 AI 商业 IP 内容 |
| 拍摄风险 | 容易讲空、讲硬、讲成工具课的地方 |

## 线索成交型脚本模板

### 模板 A：反常识纠偏

```text
如果你是企业老板，学了很多 AI 工具，但公司业务没变化，这条你一定要听完。

问题不是工具不够多，而是你一开始就用错了顺序。

老板做 AI，第一步不是学工具，而是先找公司最贵的问题。

比如获客贵、销售跟进乱、客服重复回复、内容生产慢。

正确顺序是：先找问题，再拆流程，最后选工具。

我做了一份《老板 AI 落地实战地图》，想要的评论区打“实战地图”。
```

### 模板 B：线索成交方法

```text
企业老板想通过短视频拿客户，别先想创意、热点和爆款。

你要先找能带来精准线索的成交型内容模板。

这种模板有一个特点：开头只吸引目标客户，中间直接戳业务痛点，结尾让用户留下一个明确动作。

AI 的价值不是帮你凭空想创意，而是把已经验证过的模板，替换成你的行业、产品和客户痛点。

这样你不是每天从零写文案，而是用 AI 批量生产能筛选客户的内容。

我把这个流程放进了《老板 AI 落地实战地图》，想要的评论区打“实战地图”。
```

### 模板 C：流程拆解

```text
你收藏的 AI 工具越多，公司可能越乱。

因为工具只能解决一个动作，流程才能解决一件事。

比如获客，不是用 AI 写一篇文案就结束了。

你要先拆清楚：客户从哪里来，谁来跟进，客户问了什么，最后怎么成交。

流程清楚以后，AI 才知道应该帮你做哪一步。

我把这套流程整理成《老板 AI 落地实战地图》，想要的评论区打“实战地图”。
```

## 开拍前 5 个问题

每条内容开拍前，Codex 必须判断：

1. 这个痛点是不是老板真的在意？
2. 市面上有没有类似表达跑出过评论或线索？
3. 前 3 秒是否能筛选目标客户？
4. 这条会不会只是我们自己觉得高级？
5. 用户为什么要评论“实战地图”？

如果答不清楚，不进入拍摄。

## 人和 AI 怎么分工

| 环节 | AI 负责 | 人负责 |
| --- | --- | --- |
| 找参考 | 收集和拆解市场模板 | 提供真实对标账号、行业案例 |
| 选题判断 | 判断是否有线索成交结构 | 判断是否符合业务真实情况 |
| 脚本生成 | 生成 3 个开头、主版、短版 | 判断 N哥能否自然表达 |
| 拍摄执行 | 给完整拍摄指令 | 按指令拍摄和反馈现场问题 |
| 发布承接 | 给标题、文案、评论话术 | 发布、回复、记录真实反馈 |
| 复盘 | 判断有效点和失效点 | 补充评论、私信、客户问题 |

## 当前决策

市场验证模板法成为后续选题和文案的前置步骤。以后不是先写脚本，而是先找市场证据；不是让人从零想内容，而是让 AI 拆结构、改写、派工，人负责业务判断和执行反馈。
