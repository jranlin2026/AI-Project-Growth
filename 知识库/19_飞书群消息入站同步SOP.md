# 飞书群消息入站同步SOP

## 一句话结论

飞书群机器人 Webhook 只能把 Codex 的消息发到群里，不能自动把群里的消息推回给 Codex。要让我知道阿浩、小彭在群里发了什么，需要额外建立“入站读取链路”。

## 当前已跑通的链路

本地已经具备三种能力：

```powershell
node scripts/feishu/list-chats.mjs
node scripts/feishu/list-chat-messages.mjs --hours 24
node scripts/feishu/sync-market-materials-from-chat.mjs --hours 24
```

这条链路适合 12:00 轻检查和 17:00 复盘使用：我定时读取群消息，把“视频链接、视频文案、评论区截图”整理成市场素材池，然后再拆成选题和脚本。

## 组员提交格式

阿浩、小彭只需要发三样：

```text
1. 视频链接
2. 视频文案
3. 评论区截图
```

不要求他们解释为什么值得看。判断、筛选、拆解和改写由增长负责人完成。

## 处理规则

同步到素材池后，我只做四件事：

1. 筛掉泛流量内容，只保留老板会关心的线索成交型内容。
2. 拆解开头、痛点、方法、行动指令和评论承接。
3. 选出可迁移母模板，改写成 AI 商业 IP 场景。
4. 生成次日执行包：完整脚本、拍摄要求、剪辑要求、发布文案和飞书任务。
