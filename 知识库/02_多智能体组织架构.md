# 多智能体组织架构

## 一句话结论

本项目采用“增长负责人主控 + 多个岗位智能体执行 + 共享 Markdown 文件作为事实源”的组织架构。

## 岗位设计

| 智能体 | 类比岗位 | 核心职责 | 主要文件 |
| --- | --- | --- | --- |
| 增长负责人 | CEO / 增长负责人 | 目标、策略、分工、验收、复盘 | `增长总控台.md`、`实验记录.md` |
| 内容管理智能体 | 内容主编 | 选题库、栏目、内容日历、排期 | `选题库.md`、`内容日历.md` |
| 文案智能体 | 编导 / 文案 | 标题、钩子、口播稿、CTA | `脚本库.md` |
| 剪辑智能体 | 后期导演 | 分镜、字幕、封面、B-roll、节奏 | `剪辑工作台.md` |
| 发布智能体 | 运营发布 | 平台标题、话题标签、评论承接、发布记录 | `发布记录.md` |
| 数据分析智能体 | 增长分析 | 数据记录、判断标准、复盘建议 | `数据复盘.md`、`实验记录.md` |

## 协作链路

```mermaid
flowchart LR
  A["增长负责人"] --> B["内容管理智能体"]
  B --> C["文案智能体"]
  C --> D["剪辑智能体"]
  D --> E["发布智能体"]
  E --> F["数据分析智能体"]
  F --> A
```

## 文件即事实源

聊天记录不作为最终事实源。

所有关键产出必须回写项目文件：

- 任务写入 `任务单-xxx.md`
- 选题写入 `选题库.md`
- 排期写入 `内容日历.md`
- 脚本写入 `脚本库.md`
- 剪辑说明写入 `剪辑工作台.md`
- 发布信息写入 `发布记录.md`
- 数据和判断写入 `数据复盘.md`
- 实验结论写入 `实验记录.md`

## 分工规则

- 每个智能体只改自己负责的文件。
- 增长负责人负责跨文件验收。
- 如果两个智能体需要同一信息，以任务单为准。
- 如果数据和感觉冲突，以数据分析智能体的复盘口径为准。
- 如果自动化要执行外部动作，以安全权限规则为准。
