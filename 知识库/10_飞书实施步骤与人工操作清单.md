# 飞书实施步骤与人工操作清单

## 一句话结论

Codex 可以完成飞书集成的脚本、模板、自动化提示词和数据结构设计；你需要手动完成飞书群机器人、任务清单、多维表格和开放平台应用授权。

## Codex 已完成

- 创建飞书配置模板：`feishu/.env.example`
- 创建飞书实施说明：`feishu/README.md`
- 创建角色配置：`feishu/role-map.json`
- 创建日报和视频数据字段：`feishu/daily-report-schema.md`
- 创建手动配置清单：`feishu/manual-setup-checklist.md`
- 创建飞书群 Webhook 推送脚本：`scripts/feishu/send-webhook.mjs`
- 创建每日任务生成脚本：`scripts/feishu/generate-daily-task.mjs`
- 创建飞书 OpenAPI 基础工具：`scripts/feishu/feishu-openapi.mjs`
- 创建多维表格读取脚本：`scripts/feishu/read-bitable-records.mjs`
- 升级 Codex 自动化 `ai-ip-08`，使其支持飞书推送和日报读取。

## 你需要手动完成

1. 在飞书项目群添加自定义机器人，复制 Webhook 和签名密钥。
2. 在飞书任务里创建任务清单：`AI商业IP增长项目每日作战`。
3. 在飞书多维表格创建两个表：每日作战日报、视频数据记录。
4. 在飞书开放平台创建企业自建应用，获取 `app_id` 和 `app_secret`。
5. 给应用开通多维表格读取/写入权限；后续如果自动创建任务，再开通任务权限。
6. 在项目根目录创建 `.env`，填入真实密钥和表格 ID。

## 验收顺序

第一步：测试飞书群提醒。

```powershell
node scripts/feishu/send-webhook.mjs --text "AI商业IP增长项目飞书机器人测试"
```

第二步：生成今日任务。

```powershell
node scripts/feishu/generate-daily-task.mjs
```

第三步：生成并推送今日任务。

```powershell
node scripts/feishu/generate-daily-task.mjs | node scripts/feishu/send-webhook.mjs
```

第四步：读取日报表。

```powershell
node scripts/feishu/read-bitable-records.mjs
```

## 管理架构确认

最终运行方式：

```text
增长负责人（Codex）
  -> 分配任务给内容管理、文案、剪辑、发布、数据分析智能体
  -> 智能体产出脚本、剪辑说明、发布策略、复盘口径
  -> 需要真人执行的事项进入飞书任务
  -> 组员执行后在飞书任务更新状态，并在多维表格填数据
  -> Codex 读取数据，复盘当天，生成明天任务
```
