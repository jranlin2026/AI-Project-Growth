1. 在飞书项目群添加自定义机器人，复制 Webhook 和签名密钥。
2. 在飞书任务里创建任务清单：`AI商业IP增长项目每日作战`。
3. 在飞书多维表格创建两个表：每日作战日报、视频数据记录。
4. 在飞书开放平台创建企业自建应用，获取 `app_id` 和 `app_secret`。
5. 给应用开通多维表格读取/写入权限；后续如果自动创建任务，再开通任务权限。
6. 在项目根目录创建 `.env`，填入真实密钥和表格 ID。

## 飞书任务时间戳规范

更新时间：2026-07-05

飞书任务 API 的 `due.timestamp` 必须使用 13 位毫秒级时间戳。

错误写法：

```js
String(Math.floor(date.getTime() / 1000))
```

正确写法：

```js
String(date.getTime())
```

如果传入 10 位秒级时间戳，飞书任务列表会显示成 1970 年。

创建或更新任务后必须验证：

```powershell
node scripts/feishu/list-tasklist-tasks.mjs
```

检查返回值：

```json
"due": {
  "timestamp": "1783243800000"
}
```

验收标准：

1. `due.timestamp` 是 13 位。
2. 飞书页面显示正确日期。
3. 父任务和子任务都通过验证。

详细复盘见：[[18_飞书任务时间戳故障复盘]]
