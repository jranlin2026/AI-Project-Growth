# 三人协作每日 SOP

## 目标

把 AI 商业 IP 增长项目做成“增长负责人 + 三人执行小组 + 飞书任务 + 多维表格”的每日闭环。

核心闭环：

```text
08:00 派工 -> 白天执行 -> 17:00 状态检查 -> 17:30 数据初填 -> 晚上复盘 -> 次日09:30补隔夜数据
```

## 角色分工

| 角色 | 人员 | 核心职责 | 飞书任务负责人 |
| --- | --- | --- | --- |
| 增长负责人 | Codex | 定策略、拆任务、场控进度、读数据、复盘、分配明天任务 | 不作为人工执行人 |
| 出镜负责人 | 林恩光 | 出镜拍摄、观点表达、业务判断、最终内容方向确认 | 林恩光 |
| 执行编导 A | 朱勇浩 | 拍摄协助、剪辑、发布、数据回填、脚本顺口检查、选题反馈 | 朱勇浩 |
| 执行编导 B | 彭涵奕 | 拍摄协助、剪辑、发布、数据回填、评论承接、素材整理 | 彭涵奕 |

重要修正：

朱勇浩和彭涵奕不是“一个只负责拍摄、一个只负责剪辑”。两个人都要能完成短视频执行全链路：拍摄协助、素材整理、剪辑、发布、评论承接、数据回填。每天只是由 Codex 指定谁是某条内容的主责人，谁是复核人。

## 上班时间与休息日

统一工作时间：

```text
09:00-18:00
```

固定休息日：

| 人员 | 固定休息日 | 派工规则 |
| --- | --- | --- |
| 朱勇浩 | 每周日 | 周日不分配拍摄、剪辑、发布、数据任务；脚本和选题必须在周六 17:30 前预备好，或由林总周日临时确认 |
| 彭涵奕 | 每周六 | 周六不分配拍摄、剪辑、发布、数据任务；需要她负责的内容优先在周五 17:30 前预排，或顺延到周日 |
| 林恩光 | 暂无固定休息日 | 周六/周日承担最终口径确认，避免休息人员被打扰 |

排班原则：

1. 不在固定休息日给对应人员派飞书任务。
2. 人工执行任务默认在 17:30 前完成状态更新和数据初填。
3. 21:30 不再作为人工必填节点；隔夜数据可在次日 09:30 补充。
4. 周五要提前检查周六是否有可发布素材，因为彭涵奕周六休息。
5. 周六要提前准备周日脚本/选题，因为朱勇浩周日休息。
6. 每条视频派工必须写明：主责人、复核人、是否当天发布、谁负责数据回填。

## 每日节奏

### 08:00 增长负责人派工

Codex 自动做：

1. 先读取昨日数据、日报、评论和选题库。
2. 调度选题、文案、剪辑、发布、数据分析等智能体，产出“今日执行包”。
3. Codex 作为增长负责人审定今日执行包。
4. 创建飞书父任务：`YYYY/MM/DD AI商业IP增长项目 今日作战任务`。
5. 按执行包和排班创建子任务：
   - 工作日：林总、朱勇浩、彭涵奕正常协作；朱和彭按内容主责轮换
   - 周六：不派彭涵奕；朱勇浩可承担当天拍摄协助、轻剪、预剪或素材整理
   - 周日：不派朱勇浩；彭涵奕可承担当天剪辑、发布、数据和评论承接
6. 把执行包链接、任务链接推送到飞书群。
7. 给出今日重点：必拍选题、完整脚本、拍摄时间、发布文案、评论承接、数据目标。

三个人要做：

1. 打开飞书任务清单。
2. 查看自己负责的子任务和今日执行包。
3. 按执行包执行，不从零想选题、脚本、文案和拍法。
4. 有阻塞就在对应任务评论里说明，不要只在群里口头说。

### 08:00 每日执行包

每天派工前必须先有执行包。执行包必须包含：

1. 今日增长目标。
2. 今日主选题和选题理由。
3. 完整拍摄脚本，包括标题、3 秒钩子、口播稿、CTA。
4. 拍摄安排，包括时间、版本、镜头要求、表达重点。
5. 剪辑要求，包括字幕重点、封面字、节奏和检查清单。
6. 发布安排，包括账号、发布时间、标题、发布文案、置顶评论。
7. 评论承接话术，包括关键词回复、私信引导、进群判断。
8. 数据目标和复盘重点。
9. 风险提醒。

飞书任务必须从执行包拆出来，不能只写“完成拍摄”“完成剪辑”“完成发布”。

### 09:00-12:00 脚本与拍摄准备

当日主责编导：

1. 按执行包检查今日脚本是否顺口。
2. 协助林总确认拍摄场景、镜头、素材命名。
3. 明确这条内容是否当天剪辑发布。
4. 不从零想脚本，除非 Codex 在执行包里明确要求。

当日复核编导：

1. 检查标题、钩子、口播和 CTA 是否一致。
2. 补充明日候选选题，每条只需补充标题、钩子、核心观点。

林总：

1. 按执行包拍摄指定脚本。
2. 确认脚本观点是否符合真实业务判断。
3. 对不顺口的表达可以改成自己的话，但不能改核心观点和 CTA。

Codex：

1. 输出今日脚本、标题、CTA 和拍摄口径。
2. 根据历史数据给出表达优化建议。
3. 必要时补充案例、标题和评论承接话术。

### 12:00-17:00 拍摄、剪辑、发布

林总：

1. 完成口播录制。
2. 至少录 2 个开头版本。
3. 如果临场改观点，要在任务评论里同步最终口径。

当日主责编导：

1. 按执行包剪辑指定素材。
2. 使用执行包里的标题、封面字、发布文案和置顶评论。
3. 按执行包指定账号和发布时间发布。
4. 按执行包里的评论承接话术回复。
5. 记录发布时间和发布链接。

当日复核编导：

1. 检查成片表达是否偏离核心观点。
2. 准备明天 3 条候选选题。

### 17:00 状态检查

Codex 自动做：

1. 在飞书群提醒提交进度。
2. 检查飞书任务是否更新。
3. 提醒未完成项。

三个人要做：

1. 更新自己任务状态。
2. 在任务评论里说明完成情况、风险、需要协助的地方。
3. 若视频未发布，必须说明预计完成时间。

### 17:30 数据初填

当日数据负责人：

1. 在“视频数据记录”多维表格完成初填：
   - 账号
   - 发布时间
   - 标题
   - 内容类型
   - 开头钩子
   - 播放量
   - 点赞
   - 评论
   - “实战地图”评论数
   - 私信数
   - 进群数
   - 老板型评论
   - 真实业务问题
2. 在“每日作战日报”多维表格填写今日完成情况。
3. 隔夜增长数据次日 09:30 补充。

林总：

1. 补充今天从评论、私信、社群里看到的真实业务问题。
2. 标记哪些问题值得做成明天内容。

当日复核编导：

1. 把明天选题候选写入日报。
2. 标记每个选题的目标人群和核心冲突。

### 晚上复盘

Codex 做：

1. 读取日报和视频数据。
2. 判断今天内容是否完成闭环。
3. 提炼可复用资产：
   - 高互动开头
   - 有转化潜力的评论
   - 真实业务问题
   - 可复用脚本结构
4. 生成第二天任务。

## 飞书使用规则

1. 正式派工以飞书任务为准，群消息只做提醒。
2. 每个任务必须有负责人、截止时间和完成状态。
3. 任务执行过程中的关键变更写在任务评论里。
4. 数据沉淀写入多维表格，不要只发在群里。
5. 群里只发三类消息：
   - 派工通知
   - 阻塞求助
   - 复盘结论

## 每日验收标准

当天算闭环，必须同时满足：

1. 至少 1 条视频发布。
2. 4 个飞书子任务都更新状态。
3. 视频数据记录已填写。
4. 每日作战日报已填写。
5. 至少沉淀 1 个明天可用选题。
6. 当天有一份明确的每日执行包，包含脚本、发布文案、评论承接和数据目标。

## 异常处理

| 异常 | 处理方式 |
| --- | --- |
| 林总无法拍摄 | 编导 A 当天改为图文/拆解型内容，编导 B 发布替代内容 |
| 编导 A 未交脚本 | Codex 基于选题库生成临时脚本，林总直接口播 |
| 编导 B 来不及剪辑 | 先发布轻剪辑版本，保留完播数据，精剪版次日复投 |
| 数据未回填 | 17:00 提醒；次日 09:30 不进入新任务前先补数据 |
| 内容效果差 | 不追责单条内容，复盘开头、选题、CTA、发布时间四项 |
| 周六需要剪辑/发布 | 默认使用周五预排素材；没有预排则顺延，不打扰彭涵奕休息 |
| 周日需要脚本/选题 | 默认使用周六预备稿；没有预备稿则由林总确认，不打扰朱勇浩休息 |
