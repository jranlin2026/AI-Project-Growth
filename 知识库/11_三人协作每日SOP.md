# 三人协作每日 SOP

## 目标

把 AI 商业 IP 增长项目做成“增长负责人 + 三人执行小组 + 飞书任务 + 多维表格”的每日闭环。

核心闭环：

```text
08:00 派工 -> 白天执行 -> 17:00 状态检查 -> 21:30 数据回填 -> 晚上复盘 -> 明天任务
```

## 角色分工

| 角色 | 人员 | 核心职责 | 飞书任务负责人 |
| --- | --- | --- | --- |
| 增长负责人 | Codex | 定策略、拆任务、场控进度、读数据、复盘、分配明天任务 | 不作为人工执行人 |
| 出镜负责人 | 林恩光 | 出镜拍摄、观点表达、业务判断、最终内容方向确认 | 林恩光 |
| 编导 A | 朱勇浩 | 选题、脚本、口播表达优化、明日选题储备 | 朱勇浩 |
| 编导 B | 彭涵奕 | 剪辑、发布、评论承接、私信/社群承接、视频数据记录 | 彭涵奕 |

## 每日节奏

### 08:00 增长负责人派工

Codex 自动做：

1. 创建飞书父任务：`YYYY/MM/DD AI商业IP增长项目 今日作战任务`。
2. 创建 4 个子任务：
   - 林总：完成今日出镜拍摄
   - 编导A：完成今日脚本和明日选题
   - 编导B：完成剪辑、发布、评论承接和数据记录
   - 编导B：填写视频数据记录
3. 把任务链接推送到飞书群。
4. 给出今日重点：必拍选题、核心观点、CTA、账号安排、数据目标。

三个人要做：

1. 打开飞书任务清单。
2. 查看自己负责的子任务。
3. 有阻塞就在对应任务评论里说明，不要只在群里口头说。

### 09:00-12:00 脚本与拍摄准备

编导 A：

1. 根据今日选题完成脚本。
2. 给出至少 2 个开头钩子。
3. 标明核心观点、关键案例、CTA。

林总：

1. 确认脚本观点是否符合真实业务判断。
2. 对不顺口的表达直接改成自己的话。
3. 准备拍摄。

Codex：

1. 根据脚本与历史数据给出表达优化建议。
2. 必要时补充案例、标题和 CTA。

### 12:00-17:00 拍摄、剪辑、发布

林总：

1. 完成口播录制。
2. 至少录 2 个开头版本。
3. 如果临场改观点，要在任务评论里同步最终口径。

编导 B：

1. 完成剪辑。
2. 准备标题、封面、发布文案。
3. 按账号安排发布。
4. 记录发布时间和发布链接。

编导 A：

1. 检查成片表达是否偏离核心观点。
2. 准备明天 3 条候选选题。

### 17:00 状态检查

Codex 自动做：

1. 在飞书群提醒提交进度。
2. 检查飞书任务是否更新。
3. 提醒未完成项。

三个人要做：

1. 更新自己任务状态。
2. 在任务评论里说明完成情况、风险、需要协助的地方。
3. 若视频未发布，必须说明预计完成时间。

### 21:30 数据回填

编导 B：

1. 在“视频数据记录”多维表格填写：
   - 账号
   - 发布时间
   - 标题
   - 内容类型
   - 开头钩子
   - 播放量
   - 点赞
   - 评论
   - “实战地图”评论数
   - 私信数
   - 进群数
   - 老板型评论
   - 真实业务问题
2. 在“每日作战日报”多维表格填写今日完成情况。

林总：

1. 补充今天从评论、私信、社群里看到的真实业务问题。
2. 标记哪些问题值得做成明天内容。

编导 A：

1. 把明天选题候选写入日报。
2. 标记每个选题的目标人群和核心冲突。

### 晚上复盘

Codex 做：

1. 读取日报和视频数据。
2. 判断今天内容是否完成闭环。
3. 提炼可复用资产：
   - 高互动开头
   - 有转化潜力的评论
   - 真实业务问题
   - 可复用脚本结构
4. 生成第二天任务。

## 飞书使用规则

1. 正式派工以飞书任务为准，群消息只做提醒。
2. 每个任务必须有负责人、截止时间和完成状态。
3. 任务执行过程中的关键变更写在任务评论里。
4. 数据沉淀写入多维表格，不要只发在群里。
5. 群里只发三类消息：
   - 派工通知
   - 阻塞求助
   - 复盘结论

## 每日验收标准

当天算闭环，必须同时满足：

1. 至少 1 条视频发布。
2. 4 个飞书子任务都更新状态。
3. 视频数据记录已填写。
4. 每日作战日报已填写。
5. 至少沉淀 1 个明天可用选题。

## 异常处理

| 异常 | 处理方式 |
| --- | --- |
| 林总无法拍摄 | 编导 A 当天改为图文/拆解型内容，编导 B 发布替代内容 |
| 编导 A 未交脚本 | Codex 基于选题库生成临时脚本，林总直接口播 |
| 编导 B 来不及剪辑 | 先发布轻剪辑版本，保留完播数据，精剪版次日复投 |
| 数据未回填 | 17:00 和 21:30 两次提醒；次日 08:00 不进入新任务前先补数据 |
| 内容效果差 | 不追责单条内容，复盘开头、选题、CTA、发布时间四项 |
