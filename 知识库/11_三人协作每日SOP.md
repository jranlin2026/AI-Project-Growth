# 三人协作每日 SOP

## 目标

把 AI 商业 IP 增长项目做成“增长负责人 + 三人执行小组 + 飞书任务 + 多维表格”的每日闭环。

核心闭环：

```text
08:00 派工 -> 白天执行 -> 17:00 状态检查 -> 17:30 数据初填 -> 晚上复盘 -> 次日09:30补隔夜数据
```

## 角色分工

| 角色 | 人员 | 核心职责 | 飞书任务负责人 |
| --- | --- | --- | --- |
| 增长负责人 | Codex | 定策略、拆任务、场控进度、读数据、复盘、分配明天任务 | 不作为人工执行人 |
| 出镜负责人 | 林恩光 | 出镜拍摄、观点表达、业务判断、最终内容方向确认 | 林恩光 |
| 编导 A | 朱勇浩 | 选题、脚本、口播表达优化、明日选题储备 | 朱勇浩 |
| 编导 B | 彭涵奕 | 剪辑、发布、评论承接、私信/社群承接、视频数据记录 | 彭涵奕 |

## 上班时间与休息日

统一工作时间：

```text
09:00-18:00
```

固定休息日：

| 人员 | 固定休息日 | 派工规则 |
| --- | --- | --- |
| 朱勇浩 | 每周日 | 周日不分配编导 A 任务；脚本和选题必须在周六 17:30 前预备好，或由林总周日临时确认 |
| 彭涵奕 | 每周六 | 周六不分配编导 B 任务；剪辑、发布、数据记录优先在周五 17:30 前预排，或顺延到周日 |
| 林恩光 | 暂无固定休息日 | 周六/周日承担最终口径确认，避免休息人员被打扰 |

排班原则：

1. 不在固定休息日给对应人员派飞书任务。
2. 人工执行任务默认在 17:30 前完成状态更新和数据初填。
3. 21:30 不再作为人工必填节点；隔夜数据可在次日 09:30 补充。
4. 周五要提前检查周六是否有可发布素材，因为彭涵奕周六休息。
5. 周六要提前准备周日脚本/选题，因为朱勇浩周日休息。

## 每日节奏

### 08:00 增长负责人派工

Codex 自动做：

1. 创建飞书父任务：`YYYY/MM/DD AI商业IP增长项目 今日作战任务`。
2. 按排班创建子任务：
   - 工作日：林总、编导 A、编导 B 正常协作
   - 周六：不派彭涵奕，优先使用周五预排素材或顺延
   - 周日：不派朱勇浩，使用周六预备稿或由林总临时确认
3. 把任务链接推送到飞书群。
4. 给出今日重点：必拍选题、核心观点、CTA、账号安排、数据目标。

三个人要做：

1. 打开飞书任务清单。
2. 查看自己负责的子任务。
3. 有阻塞就在对应任务评论里说明，不要只在群里口头说。

### 09:00-12:00 脚本与拍摄准备

编导 A：

1. 根据今日选题完成脚本。
2. 给出至少 2 个开头钩子。
3. 标明核心观点、关键案例、CTA。

林总：

1. 确认脚本观点是否符合真实业务判断。
2. 对不顺口的表达直接改成自己的话。
3. 准备拍摄。

Codex：

1. 根据脚本与历史数据给出表达优化建议。
2. 必要时补充案例、标题和 CTA。

### 12:00-17:00 拍摄、剪辑、发布

林总：

1. 完成口播录制。
2. 至少录 2 个开头版本。
3. 如果临场改观点，要在任务评论里同步最终口径。

编导 B：

1. 完成剪辑。
2. 准备标题、封面、发布文案。
3. 按账号安排发布。
4. 记录发布时间和发布链接。

编导 A：

1. 检查成片表达是否偏离核心观点。
2. 准备明天 3 条候选选题。

### 17:00 状态检查

Codex 自动做：

1. 在飞书群提醒提交进度。
2. 检查飞书任务是否更新。
3. 提醒未完成项。

三个人要做：

1. 更新自己任务状态。
2. 在任务评论里说明完成情况、风险、需要协助的地方。
3. 若视频未发布，必须说明预计完成时间。

### 17:30 数据初填

编导 B：

1. 在“视频数据记录”多维表格完成初填：
   - 账号
   - 发布时间
   - 标题
   - 内容类型
   - 开头钩子
   - 播放量
   - 点赞
   - 评论
   - “实战地图”评论数
   - 私信数
   - 进群数
   - 老板型评论
   - 真实业务问题
2. 在“每日作战日报”多维表格填写今日完成情况。
3. 隔夜增长数据次日 09:30 补充。

林总：

1. 补充今天从评论、私信、社群里看到的真实业务问题。
2. 标记哪些问题值得做成明天内容。

编导 A：

1. 把明天选题候选写入日报。
2. 标记每个选题的目标人群和核心冲突。

### 晚上复盘

Codex 做：

1. 读取日报和视频数据。
2. 判断今天内容是否完成闭环。
3. 提炼可复用资产：
   - 高互动开头
   - 有转化潜力的评论
   - 真实业务问题
   - 可复用脚本结构
4. 生成第二天任务。

## 飞书使用规则

1. 正式派工以飞书任务为准，群消息只做提醒。
2. 每个任务必须有负责人、截止时间和完成状态。
3. 任务执行过程中的关键变更写在任务评论里。
4. 数据沉淀写入多维表格，不要只发在群里。
5. 群里只发三类消息：
   - 派工通知
   - 阻塞求助
   - 复盘结论

## 每日验收标准

当天算闭环，必须同时满足：

1. 至少 1 条视频发布。
2. 4 个飞书子任务都更新状态。
3. 视频数据记录已填写。
4. 每日作战日报已填写。
5. 至少沉淀 1 个明天可用选题。

## 异常处理

| 异常 | 处理方式 |
| --- | --- |
| 林总无法拍摄 | 编导 A 当天改为图文/拆解型内容，编导 B 发布替代内容 |
| 编导 A 未交脚本 | Codex 基于选题库生成临时脚本，林总直接口播 |
| 编导 B 来不及剪辑 | 先发布轻剪辑版本，保留完播数据，精剪版次日复投 |
| 数据未回填 | 17:00 提醒；次日 09:30 不进入新任务前先补数据 |
| 内容效果差 | 不追责单条内容，复盘开头、选题、CTA、发布时间四项 |
| 周六需要剪辑/发布 | 默认使用周五预排素材；没有预排则顺延，不打扰彭涵奕休息 |
| 周日需要脚本/选题 | 默认使用周六预备稿；没有预备稿则由林总确认，不打扰朱勇浩休息 |
