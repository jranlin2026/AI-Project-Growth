# 市场素材提交 SOP

## 一句话结论

第一阶段不强求 Codex 直接自动刷抖音。市场素材由人负责发现和搬运，Codex 负责判断、拆解、改写和派工。阿浩、小彭不需要解释为什么值得看，只需要按固定格式提交素材。

## 为什么这么做

抖音、小红书、视频号等平台对自动化浏览器读取不稳定，直接让 AI 控浏览器做调研会卡在技术细节上。

当前更高效的方式是：

```text
人刷平台更快
  -> 把素材丢到飞书群
  -> Codex 判断是否值得拆
  -> Codex 改写成今日选题和拍摄脚本
  -> 飞书任务派工执行
```

这套流程先保证增长闭环跑起来，后续再逐步自动化平台调研。

## 人只提交三样东西

提交格式固定为：

```text
【市场素材】
1. 视频链接：
2. 视频文案：
3. 评论区截图：已发/见图
```

如果视频文案太长，可以直接截图或粘贴核心口播。

不要求提交：

1. 为什么值得看。
2. 适合改成什么方向。
3. 选题分析。
4. 爆款原因。
5. 用户画像判断。

这些都由 Codex 增长负责人判断。

不用写为什么值得看，也不用分析。判断和改写由 Codex 增长负责人来做。

验收标准：
- 群里能看到链接、文案、评论截图

## Codex 如何处理素材

Codex 每天 08:00 或复盘时做判断：

1. 先筛掉泛流量内容，只保留老板会关心的内容。
2. 看开头是否能筛选精准人群。
3. 看文案是否有线索成交结构。
4. 看评论区是否出现真实需求，例如：
   - 怎么做？
   - 资料怎么领？
   - 适合什么行业？
   - 老板自己怎么落地？
   - 团队怎么执行？
5. 判断能否迁移到 AI 商业 IP 场景。
6. 选出 1 条母模板，改写成今日执行包。

## 2026-07-05 入站同步补充

今天发现一个关键边界：群机器人 Webhook 只能发送消息，不能读取群成员发的消息。所以组员在群里发了链接、文案、截图以后，Codex 不会自动知道。

已新增入站同步机制：

```powershell
node scripts/feishu/list-chats.mjs
node scripts/feishu/list-chat-messages.mjs --hours 24
node scripts/feishu/sync-market-materials-from-chat.mjs --hours 24
```

新的标准流程：

```text
阿浩/小彭在群里发三件套
  -> Codex 在 12:00 或 17:00 读取群消息
  -> 自动生成市场素材同步文件
  -> 增长负责人拆解素材
  -> 进入次日选题、脚本和执行包
```

前置条件：

1. 飞书开放平台应用开通群聊读取权限。
2. 飞书开放平台应用开通消息读取权限。
3. `.env` 配置 `FEISHU_MARKET_MATERIAL_CHAT_ID`。

这条机制跑通后，组员只需要按三件套提交，不需要再单独提醒 Codex“我发了”。
