# AI 与人协同增长 SOP

## 一句话结论

AI 负责市场验证、选题判断、脚本生成、任务拆解、过程提醒和数据复盘；人负责真实业务判断、拍摄执行、剪辑发布、评论承接和现场反馈。协作目标不是“让人听 AI 指挥”，而是让人只做 AI 不能替代的高价值动作。

## 核心分工

| 角色 | 核心价值 | 不该做什么 |
| --- | --- | --- |
| Codex 增长负责人 | 判断方向、拆任务、写脚本、看数据、调第二天策略 | 等人给模糊指令、只发笼统任务 |
| N哥 | 出镜表达、业务判断、信任建立、最终口径确认 | 每天从零想选题和完整脚本 |
| 阿浩 | 拍摄协助、脚本顺口检查、素材整理、轻剪预处理、选题反馈 | 脱离执行包从零写大方向 |
| 小彭 | 剪辑、发布、评论承接、数据回填、素材整理 | 在休息日被临时拉任务 |

## 协作总流程

```text
市场验证模板
  -> Codex 生成今日执行包
  -> 飞书群发安排摘要 + 完整脚本文档
  -> 飞书任务拆到人
  -> 人执行拍摄/剪辑/发布
  -> 12:00 轻检查
  -> 17:00 状态检查
  -> 17:30 最低反馈
  -> Codex 复盘并决定明天
```

## 08:00：AI 派工

08:00 必须产出四样东西。

### 1. 今日执行包

必须包含：

1. 今日目标。
2. 今日选题。
3. 市场验证依据。
4. 完整主版脚本。
5. 备用短版脚本。
6. 拍摄口语提示。
7. 剪辑要求。
8. 发布文案。
9. 评论承接话术。
10. 数据目标。
11. 人员任务拆分。

### 2. 飞书群安排摘要

飞书群消息要像负责人正常说话。

错误风格：

```text
请各负责人根据任务完成今日拍摄、剪辑、发布、数据记录。
```

正确风格：

```text
今天先把 S002 跑完，不重新想新题。
N哥确认素材，小彭主责剪辑发布，阿浩今天休息不安排任务。
这条重点别剪成硬广，要保留聊天感。
```

### 3. 飞书群完整脚本文档

群里必须发完整脚本文档，不能只发摘要或链接。

完整脚本文档至少包含：

1. 主题。
2. 这条视频要让用户听懂什么。
3. 开头 3 个版本。
4. 主版口播稿。
5. 备用短版。
6. N哥拍摄提醒。
7. 阿浩或小彭执行提醒。

### 4. 飞书正式任务

任务必须拆到人，不能只写岗位。

任务字段必须包含：

1. 负责人。
2. 截止时间。
3. 完整交付物。
4. 素材命名。
5. 是否需要发布。
6. 是否需要数据回填。
7. 如果卡住，在哪里反馈。

## 09:00-12:00：人执行前半段

N哥：

1. 按脚本文档确认观点是否符合真实业务。
2. 不用背稿，可以按自己的话说。
3. 不能改核心观点和 CTA。

当日主责：

1. 拍前看脚本是否顺口。
2. 帮 N哥盯拍摄重点。
3. 确认素材命名。
4. 如果脚本不顺，直接在任务评论或群里说。

Codex：

1. 如果 12:00 前有人反馈脚本不顺，马上改稿。
2. 如果素材未开始，提醒补救。
3. 如果今天任务不存在，立刻补建任务。

## 12:00：轻检查

12:00 不要求填完整数据，只问执行状态。

最低反馈格式：

```text
已开始 / 未开始 / 已完成 / 卡在哪里 / 需要什么支持
```

飞书群提醒口吻：

```text
12点看一下进度。
N哥这边素材拍了吗？
主责这边素材拿到了没？
如果卡在脚本或拍摄，直接说，我现在改。
```

## 17:00：状态检查

Codex 做：

1. 发飞书群提醒。
2. 读取飞书任务。
3. 读取多维表格。
4. 如果表格没有数据，也检查任务评论和群反馈。
5. 点名缺失项。

人要回：

```text
【今日结果】
任务：
完成情况：
素材/链接：
遇到问题：
明天建议：
```

如果已经发布，再补：

```text
播放：
点赞：
评论：
实战地图评论：
私信：
进群：
老板型问题：
```

## 17:30：最低闭环标准

当天不一定必须多维表格填满，但至少要有可判断的反馈。

最低闭环：

1. 飞书任务状态或任务评论有更新。
2. 群里有人明确回复完成情况。
3. 素材或发布链接可追踪。
4. 如果发布，至少有初始数据。
5. 如果未发布，必须说明原因和补救时间。

多维表格用于长期复盘，不是唯一判断来源。

## 复盘规则

Codex 复盘不能只写“已完成/未完成”，必须做决策。

每天复盘输出：

1. 今天有没有闭环。
2. 哪个环节卡住。
3. 内容方向是否继续。
4. 哪个开头值得复用。
5. 评论里有没有真实业务问题。
6. 明天拍什么。
7. 明天谁主责，谁休息。

## AI 和人的边界

AI 可以直接做：

1. 找市场模板。
2. 拆选题结构。
3. 写脚本。
4. 写飞书任务。
5. 写群消息。
6. 写发布文案。
7. 写评论话术。
8. 做数据复盘。
9. 调整第二天任务。

人必须参与：

1. 真实业务判断。
2. 出镜拍摄。
3. 剪辑审美和成片执行。
4. 平台发布。
5. 评论和私信真实承接。
6. 客户问题反馈。
7. 对 AI 输出提出“不像人话”“不适合拍”的纠偏。

## 当前阶段重点

现阶段不要追求云端自动化，也不要追求智能体全自动。

当前最重要的是：

```text
市场验证选题
  -> 能拍的口语脚本
  -> 明确到人的执行
  -> 最低反馈闭环
  -> 根据真实数据改第二天
```

等这个闭环连续跑 7-14 天，再把各角色智能体进一步训练和自动化。
