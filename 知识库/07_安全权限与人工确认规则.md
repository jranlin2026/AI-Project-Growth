# 安全权限与人工确认规则

## 一句话结论

智能体可以自动做低风险、可撤回、可检查的动作；涉及账号安全、公开发布、客户沟通、资金和删除修改的动作，必须人工确认。

## 可自动执行

| 动作 | 条件 |
| --- | --- |
| 读取项目文件 | 只读或按角色文件范围读取 |
| 写脚本、剪辑说明、发布文案 | 写入对应项目文件 |
| 整理内容日历 | 不删除历史记录 |
| 建立发布清单 | 不直接发布 |
| 上传视频并保存草稿 | 平台状态稳定，文件明确 |
| 查询公开数据 | 不涉及敏感信息 |
| 生成日报和复盘 | 基于已有数据 |

## 需要人工确认

| 动作 | 原因 |
| --- | --- |
| 最终发布视频 | 对外品牌动作 |
| 删除视频或评论 | 不可逆或影响账号 |
| 修改账号资料 | 影响账号定位 |
| 回复高意向客户 | 影响成交和信任 |
| 私信销售转化 | 需要真人判断语气和承诺 |
| 投流、充值、支付 | 涉及资金 |
| 处理验证码、人脸、短信 | 涉及账号安全 |
| 导出或上传客户资料 | 涉及隐私和合规 |

## 浏览器/电脑控制规则

智能体控制浏览器或电脑时，必须遵守：

- 操作前说明目标。
- 只打开和任务有关的网站或软件。
- 遇到登录、验证码、短信、人脸验证时停止。
- 不保存、展示、记录密码。
- 不修改账号安全设置。
- 不删除文件、视频、评论或私信。
- 完成后回写项目文件。

## 发布权限分级

| 级别 | 权限 | 说明 |
| --- | --- | --- |
| P0 | 只读 | 只查看文件和网页 |
| P1 | 写项目文件 | 可更新 Markdown 台账 |
| P2 | 准备草稿 | 可上传视频并保存草稿 |
| P3 | 人工确认后发布 | 你确认后由智能体点击发布 |
| P4 | 自动发布 | 仅在连续稳定后开放 |

## 默认策略

当前默认权限建议：

- 增长负责人：P1
- 内容管理智能体：P1
- 文案智能体：P1
- 剪辑智能体：P1
- 发布智能体：P2
- 数据分析智能体：P1

发布智能体暂不开放 P4。
