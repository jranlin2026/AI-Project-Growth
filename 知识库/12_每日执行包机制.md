# 每日执行包机制

## 一句话结论

飞书任务不能只写“拍摄、剪辑、发布、数据记录”，必须先由 Codex 增长负责人产出“今日执行包”，再把执行包拆成飞书任务。三个人不负责从零想选题、脚本、文案和拍法，只负责按明确内容执行、反馈和补充真实业务判断。

## 正确流程

```text
昨日数据/日报
  -> Codex 增长负责人判断今天目标
  -> 调度选题/文案/发布/数据分析智能体产出执行包
  -> Codex 审定执行包
  -> 飞书任务派给三个人执行
  -> 17:00 检查状态
  -> 17:30 数据初填
  -> 晚上复盘生成明天执行包
```

## 每日执行包必须包含

每天 08:00 派工前，Codex 必须先产出以下内容：

| 模块 | 必填内容 | 负责人 |
| --- | --- | --- |
| 今日增长目标 | 今天要验证什么、目标数据是什么、为什么今天做这个 | Codex |
| 今日选题 | 选题 ID、栏目、目标账号、目标痛点、为什么优先 | 选题/数据分析智能体 + Codex |
| 拍摄脚本 | 标题、3 秒钩子、完整口播稿、备选开头、CTA | 文案智能体 + Codex |
| 拍摄安排 | 拍摄时间、镜头要求、必须录几版、每版差异 | Codex |
| 剪辑要求 | 字幕重点、节奏、封面字、需要强调的画面/关键词 | 剪辑智能体 + Codex |
| 发布安排 | 账号、发布时间、标题、发布文案、评论置顶内容 | 发布智能体 + Codex |
| 评论承接 | 关键词回复、私信引导、进群话术、老板型线索判断 | 内容管理智能体 + Codex |
| 数据分析 | 今天重点看哪些指标、达到什么算有效、失败看什么 | 数据分析智能体 + Codex |
| 风险提醒 | 哪些话不能说、哪些口径不能偏、哪里需要人工确认 | Codex |

## 飞书任务拆法

飞书任务必须从执行包拆出来，不能只写岗位职责。

### N哥任务必须写清楚

必须包含：

1. 今天拍哪条脚本。
2. 完整口播稿。
3. 至少 2 个开头版本。
4. 录制时间。
5. 表达重点。
6. 不能偏离的口径。

错误写法：

```text
N哥：完成今日出镜拍摄
```

正确写法：

```text
N哥：14:00-15:00 拍 S001《老板做 AI，最不该先学工具》
口播稿：……
开头版本 A：……
开头版本 B：……
表达重点：别讲工具清单，讲业务问题优先级。
```

### 编导 A 任务必须写清楚

编导 A 不从零想脚本，除非 Codex 明确要求。

必须包含：

1. 今天要审核哪条脚本。
2. 需要检查哪些表达。
3. 要准备明天哪 3 条备选选题。
4. 每条备选选题的钩子和核心观点。

错误写法：

```text
编导A：完成今日脚本和明日选题
```

正确写法：

```text
编导A：12:00 前检查 S001 口播稿是否顺口；17:30 前准备明天 3 条候选：
T002《别再收藏 AI 工具了》
T003《为什么你的视频没有客户》
T005《销售不是不努力，是没有 AI 跟进系统》
每条必须给 1 个标题、1 个 3 秒钩子、1 句核心观点。
```

### 编导 B 任务必须写清楚

必须包含：

1. 剪哪条素材。
2. 发布到哪个账号。
3. 标题用哪个。
4. 发布文案是什么。
5. 置顶评论是什么。
6. 评论区关键词怎么回复。
7. 数据记录字段。

错误写法：

```text
编导B：完成剪辑、发布、评论承接和数据记录
```

正确写法：

```text
编导B：剪辑并发布 S001 到账号 B
标题：老板做 AI，最不该先学工具
发布文案：……
置顶评论：想要《老板 AI 落地实战地图》的，评论区打“实战地图”
评论回复：……
数据字段：播放、点赞、评论、实战地图评论数、私信、进群、老板型线索。
```

## 智能体使用方式

Codex 增长负责人每天负责调度其他智能体：

| 智能体 | 每天输入 | 每天输出 |
| --- | --- | --- |
| 选题智能体 | 昨日数据、选题库、评论问题 | 今日主选题、备选选题、选题理由 |
| 文案智能体 | 今日主选题、目标人群、CTA | 完整脚本、开头钩子、标题、拍摄口径 |
| 剪辑智能体 | 脚本、素材要求、发布账号 | 剪辑节奏、字幕重点、封面字、成片检查清单 |
| 发布智能体 | 成片、账号安排、发布时间 | 发布标题、发布文案、置顶评论、评论承接话术 |
| 数据分析智能体 | 多维表格数据、评论、私信 | 今日复盘、有效选题、失效原因、明天建议 |
| 内容管理智能体 | 评论区、社群、私信反馈 | 真实业务问题、老板型线索、二创选题 |

人类成员只做三件事：

1. 执行已经定好的内容。
2. 反馈真实业务判断和现场情况。
3. 把执行结果和数据填回系统。

## 每天 08:00 输出格式

Codex 每天 08:00 必须输出：

```text
今日目标：
今日主选题：
为什么今天做它：
拍摄脚本：
拍摄安排：
剪辑要求：
发布安排：
评论承接：
数据目标：
飞书任务拆分：
风险提醒：
```

如果缺少昨日数据，Codex 也不能只派岗位任务，必须基于当前冷启动计划直接给出明确执行包。
