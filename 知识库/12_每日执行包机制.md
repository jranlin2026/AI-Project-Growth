# 每日执行包机制

## 一句话结论

飞书任务不能只写“拍摄、剪辑、发布、数据记录”，必须先由 Codex 增长负责人基于市场验证模板产出“今日执行包”，再把执行包拆成飞书任务。三个人不负责从零想选题、脚本、文案和拍法，只负责按明确内容执行、反馈和补充真实业务判断。

## 正确流程

```text
飞书群市场素材/市场验证模板/昨日数据/日报/评论反馈
  -> Codex 增长负责人判断今天目标
  -> 调度选题/文案/剪辑/发布/数据分析智能体产出执行包
  -> Codex 审定执行包
  -> 飞书群发送今日安排摘要和完整拍摄脚本文档
  -> 飞书任务派给三个人执行
  -> 12:00 轻检查
  -> 17:00 检查状态
  -> 17:30 最低反馈和数据初填
  -> 晚上复盘生成明天执行包
```

## 每日执行包必须包含

每天 08:00 派工前，Codex 必须先产出以下内容：

| 模块 | 必填内容 | 负责人 |
| --- | --- | --- |
| 今日增长目标 | 今天要验证什么、目标数据是什么、为什么今天做这个 | Codex |
| 市场验证依据 | 参考了哪条市场素材、哪类爆款模板、评论区是否有线索、可复用的获客结构是什么 | 选题/数据分析智能体 + Codex |
| 今日选题 | 选题 ID、栏目、目标账号、目标痛点、为什么优先 | 选题/数据分析智能体 + Codex |
| 拍摄脚本 | 标题、3 秒钩子、完整口播稿、短版口播稿、备选开头、CTA | 文案智能体 + Codex |
| 拍摄安排 | 拍摄时间、镜头要求、必须录几版、每版差异 | Codex |
| 剪辑要求 | 字幕重点、节奏、封面字、需要强调的画面/关键词 | 剪辑智能体 + Codex |
| 发布安排 | 账号、发布时间、标题、发布文案、评论置顶内容 | 发布智能体 + Codex |
| 评论承接 | 关键词回复、私信引导、进群话术、老板型线索判断 | 内容管理智能体 + Codex |
| 数据分析 | 今天重点看哪些指标、达到什么算有效、失败看什么 | 数据分析智能体 + Codex |
| 风险提醒 | 哪些话不能说、哪些口径不能偏、哪里需要人工确认 | Codex |

## 飞书任务拆法

飞书任务必须从执行包拆出来，不能只写岗位职责。

### 群消息必须先说人话

飞书群消息不是公告，不写成“请各负责人完成今日工作”。群里只讲三件事：

1. 今天拍哪条，为什么拍。
2. N哥、阿浩、小彭分别要交什么。
3. 17:30 前最低要反馈什么。

群里还必须单独发一份“完整拍摄脚本文档”，让执行人员可以直接照着看，不需要在任务详情里到处找。

### N哥任务必须写清楚

必须包含：

1. 今天拍哪条脚本。
2. 完整口播稿。
3. 至少 2 个开头版本。
4. 录制时间。
5. 表达重点。
6. 不能偏离的口径。

错误写法：

```text
N哥：完成今日出镜拍摄
```

正确写法：

```text
N哥：14:00-15:00 拍 S001《老板做 AI，最不该先学工具》
口播稿：……
开头版本 A：……
开头版本 B：……
表达重点：别讲工具清单，讲业务问题优先级。
```

### 当日主责编导任务必须写清楚

阿浩和小彭都要能负责拍摄协助、剪辑、发布、评论承接和数据回填。每天由 Codex 指定谁是主责，谁是复核。主责不从零想脚本，除非 Codex 明确要求。

必须包含：

1. 今天跟进哪条视频。
2. 拍摄协助要检查哪些表达和镜头。
3. 剪辑用哪个标题、封面字、字幕重点。
4. 发布用哪个账号、标题、文案和置顶评论。
5. 评论区关键词怎么回复。
6. 数据记录字段和最低反馈格式。

错误写法：

```text
阿浩：完成今日拍摄剪辑发布
```

正确写法：

```text
阿浩：今天主责 S001
10:30 前帮 N哥确认脚本是否顺口，14:00 协助拍摄。
剪辑标题：老板做 AI，最不该先学工具
封面字：老板做 AI，别先学工具
置顶评论：想要《老板AI落地实战地图》的，评论区打“实战地图”
17:30 前反馈：是否发布、视频链接、初始播放/评论/私信、卡点。
```

### 当日复核编导任务必须写清楚

必须包含：

1. 检查成片是否偏离核心观点。
2. 检查标题、封面、字幕、CTA 是否统一。
3. 准备明天候选选题。
4. 记录评论区真实业务问题。

错误写法：

```text
小彭：完成复核和选题
```

正确写法：

```text
小彭：今天复核 S001
发布前检查：前 3 秒是否能筛选企业老板、字幕是否突出“别先学工具”、CTA 是否指向实战地图。
17:30 前补 3 条明天候选：每条写标题、3 秒钩子、核心观点、适合参考的市场模板。
```

## 智能体使用方式

Codex 增长负责人每天负责调度其他智能体：

| 智能体 | 每天输入 | 每天输出 |
| --- | --- | --- |
| 选题智能体 | 昨日数据、选题库、评论问题 | 今日主选题、备选选题、选题理由 |
| 文案智能体 | 今日主选题、目标人群、CTA | 完整脚本、开头钩子、标题、拍摄口径 |
| 剪辑智能体 | 脚本、素材要求、发布账号 | 剪辑节奏、字幕重点、封面字、成片检查清单 |
| 发布智能体 | 成片、账号安排、发布时间 | 发布标题、发布文案、置顶评论、评论承接话术 |
| 数据分析智能体 | 多维表格数据、评论、私信 | 今日复盘、有效选题、失效原因、明天建议 |
| 内容管理智能体 | 评论区、社群、私信反馈 | 真实业务问题、老板型线索、二创选题 |

人类成员只做三件事：

1. 执行已经定好的内容。
2. 反馈真实业务判断和现场情况。
3. 把执行结果和数据填回系统。

市场素材提交属于“人类成员发现素材，Codex 判断价值”的前置动作。阿浩和小彭只需要按 [[16_市场素材提交SOP]] 提交链接、文案和评论区截图，不需要做选题判断。

## 每天 08:00 输出格式

Codex 每天 08:00 必须输出：

```text
今日目标：
市场验证依据：
参考市场素材：
今日主选题：
为什么今天做它：
拍摄脚本：
短版脚本：
拍摄安排：
剪辑要求：
发布安排：
评论承接：
数据目标：
飞书任务拆分：
风险提醒：
```

如果缺少昨日数据，Codex 也不能只派岗位任务，必须基于当前冷启动计划直接给出明确执行包。

## 每天 12:00 轻检查

12:00 只做轻检查，不要求三个人完整填表。

Codex 要自然地在群里确认：

```text
今天这条开始动了吗？
N哥脚本有没有不顺口的地方？
阿浩/小彭谁这边卡住了？
如果素材还没拍，我现在直接改成更容易拍的版本。
```

三个人只需要反馈：

```text
已开始 / 未开始 / 已完成 / 卡在哪里 / 需要什么支持
```

## 数据不是唯一反馈入口

多维表格是长期复盘资产，不是唯一判断“做没做”的入口。

当天执行状态可以来自三类地方：

1. 飞书任务状态和任务评论。
2. 飞书群里的明确反馈。
3. 多维表格里的日报和视频数据。

但到次日 09:30 前，发布过的视频必须补齐多维表格，因为后续选题判断、脚本优化和数据分析都要靠结构化数据。
