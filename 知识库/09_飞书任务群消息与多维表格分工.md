# 飞书任务、群消息与多维表格分工

## 一句话结论

飞书任务负责“谁今天要完成什么”，群消息负责“提醒大家看任务和交日报”，多维表格负责“沉淀日报、视频数据和复盘依据”。

## 推荐组合

不要只用群消息，也不要只用飞书任务。

最佳组合：

```text
飞书任务 = 派工与状态追踪
飞书群消息 = 通知与提醒
飞书多维表格 = 日报、视频数据、线索数据和复盘数据库
Codex = 生成任务、分析数据、安排明天
```

## 为什么不能只用群消息

群消息适合广播，但不适合管理任务。

问题：

- 消息容易被刷走
- 谁负责什么不够清晰
- 完成状态不好追踪
- 历史任务不好汇总
- 组员容易只回复“收到”，但没有交付

## 为什么要用飞书任务

飞书任务适合作为正式派工系统。

适合管理：

- 负责人
- 截止时间
- 子任务
- 完成状态
- 任务评论
- 今日待办
- 我负责的任务

## 为什么还需要多维表格

飞书任务适合管理执行，但不适合作为增长数据仓库。

短视频增长项目需要结构化记录：

- 每条视频播放量
- 点赞数
- 评论数
- “实战地图”评论数
- 私信数
- 进群人数
- 老板型线索
- 真实业务问题
- 最好视频和最差视频
- 每日复盘结论

这些数据更适合放在多维表格，方便 Codex 读取、筛选、统计和复盘。

## 第一版工作流

1. Codex 08:00 生成今日作战计划。
2. 系统在飞书任务里创建当天任务和子任务。
3. 飞书群发一条提醒：今日任务已创建，请查看任务清单。
4. 组员白天执行任务。
5. 17:00 飞书群提醒提交日报。
6. 组员在多维表格填写完成情况和视频数据。
7. Codex 读取多维表格，分析完成情况。
8. Codex 生成第二天任务。

## 任务拆分方式

每天创建一个父任务：

```text
2026-07-03 AI商业IP增长项目 Day X 作战任务
```

父任务下创建子任务：

| 子任务 | 负责人 | 截止时间 |
| --- | --- | --- |
| 林总：完成今日出镜拍摄 | 林总 | 17:00 |
| 编导 A：完成今日脚本和明日选题 | 编导 A | 17:00 |
| 编导 B：完成剪辑、发布、评论承接和数据记录 | 编导 B | 17:30 |
| 数据复盘：填写视频数据表初填 | 编导 B / 数据分析智能体 | 17:30 |

## 群消息模板

### 08:00

```text
今日作战任务已创建。

请各负责人进入飞书任务查看自己负责的事项：
- 林总：出镜拍摄任务
- 编导 A：脚本与选题任务
- 编导 B：剪辑、发布、评论承接、数据记录任务

17:00 前必须更新任务状态，并填写日报表。
```

### 17:00

```text
请提交今日日报。

要求：
1. 更新飞书任务状态
2. 在多维表格填写完成情况
3. 填写每条视频数据
4. 标记老板型评论和真实业务问题

未填写视为今日任务未闭环。
```

## 是否可以不要多维表格

如果只是管理“今天谁做什么”，可以不用多维表格。

但本项目是增长项目，需要长期复盘内容、账号、钩子、线索和成交机会，因此多维表格必须保留。

结论：

> 飞书任务管执行，多维表格管数据，群消息管提醒。
