# 发布智能体自动化 SOP

## 一句话结论

发布智能体未来的核心任务，是从 `发布记录.md` 读取待发布内容，控制浏览器或电脑完成上传和草稿准备，再把状态回写到项目文件。

## 输入文件

发布智能体每次执行前必须读取：

- `任务单-xxx.md`
- `内容日历.md`
- `脚本库.md`
- `剪辑工作台.md`
- `发布记录.md`

## 输出文件

发布后必须更新：

- `发布记录.md`
- 必要时更新 `数据复盘.md` 的待上线状态

## 半自动发布流程

1. 读取 `发布记录.md` 中状态为“待成片/待发布”的内容。
2. 确认脚本 ID、账号、平台、标题、话题标签。
3. 确认本地视频文件路径。
4. 打开平台后台。
5. 上传视频。
6. 填入标题。
7. 填入话题标签。
8. 选择封面或保留默认封面。
9. 检查评论承接话术是否已准备。
10. 保存草稿。
11. 回写发布记录：草稿已创建、时间、平台、账号、待人工确认。

## 不直接发布的默认策略

在系统稳定前，发布智能体默认只保存草稿，不直接发布。

原因：

- 账号发布动作有品牌风险
- 平台后台可能有临时弹窗或风控
- 标题、封面、视频文件可能需要人工最终确认
- 初期要通过人工确认训练发布质量

## 允许直接发布的前提

同时满足以下条件后，才可以考虑开放自动发布：

- 连续 10 次保存草稿无错误
- 标题、话题标签和封面检查通过率稳定
- 平台登录状态稳定
- 没有触发验证码或风控
- 增长负责人明确授权

## 发布后回写字段

| 字段 | 内容 |
| --- | --- |
| 日期 | 实际发布日期 |
| 平台 | 抖音 / 视频号 / 小红书等 |
| 账号 | 账号 A/B/C |
| 脚本 ID | 例如 S001 |
| 平台标题 | 实际发布标题 |
| 话题标签 | 实际使用标签 |
| 发布时间 | 实际发布时间 |
| 链接 | 视频链接或草稿状态 |
| 评论承接状态 | 待承接 / 已承接 / 有高意向 |
| 数据记录状态 | 待记录 / 已记录 |

## 异常处理

| 异常 | 处理方式 |
| --- | --- |
| 未登录 | 停止操作，通知人工登录 |
| 验证码 / 短信 / 人脸 | 停止操作，等待人工处理 |
| 找不到视频文件 | 回写缺失原因，不继续发布 |
| 平台弹窗 | 截图或描述弹窗，等待确认 |
| 上传失败 | 重试 1 次，仍失败则停止 |
| 标题超长 | 缩短标题后保存草稿，不直接发布 |
